# Project Architecture

## GitNerd V1

# 1. Architecture Overview

GitNerd is organized into independent modules, where each module has a single responsibility. This modular architecture makes the project easier to develop, maintain, and extend.

```text
                    GitNerd
                        │
          ┌─────────────┼─────────────┐
          │             │             │
          ▼             ▼             ▼
     Frontend      Backend      Documentation
                        │
            ┌───────────┴─────────────────────────────┐
            │                       │                 │
            ▼                       ▼                 ▼
       AI Engine               Database             (Others)
```

# 7. Naming Conventions

* Use `snake_case` for Python files.
* Use `PascalCase` for classes.
* Use `camelCase` only when required by external libraries.
* Follow the principle of one responsibility per module.
* Use meaningful and descriptive names for files, classes, and functions.
* Keep modules independent whenever possible.

# 8. Development Guidelines

* Design before implementation.
* Keep the architecture modular.
* The backend coordinates all application requests.
* Keep AI logic inside the AI Engine.
* Treat the Repository Knowledge Base as the single source of truth.
* Update documentation whenever the architecture changes.
* Build one milestone at a time.
* Commit one completed feature at a time.
* Refactor only after the feature works correctly.
* Prefer simplicity over unnecessary complexity.
